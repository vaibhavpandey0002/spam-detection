## Slide 2: Project Overview & Objectives
- **Problem**: Spam messages and emails are a major annoyance and security threat (phishing, scams).
- **Solution**: An automated ML pipeline to classify incoming text as "Spam" or "Ham" (Not Spam).
- **Objectives**:
  - Build a highly accurate NLP classification model.
  - Expose the model via a REST API (FastAPI).
  - Create an intuitive, modern UI for real-time predictions.

## Slide 3: The Machine Learning Pipeline
- *Include the simple pipeline diagram here*
- **Input Text** -> **Preprocessing** -> **Feature Extraction** -> **Model Prediction** -> **Output**
- Emphasize that the system is end-to-end, meaning it handles everything from raw text input to user-friendly visual output.

## Slide 4: Dataset & Preprocessing
- **Dataset**: UCI SMS Spam Collection (5,500+ messages labeled as spam/ham).
- **Preprocessing Steps (NLTK)**:
  1. Lowercasing
  2. Removing Punctuation
  3. Tokenization (splitting into words)
  4. Stopword Removal (removing common words like 'the', 'is', 'and')

## Slide 5: Feature Extraction (TF-IDF)
- **Problem**: Machine learning models cannot understand raw text.
- **Solution**: TF-IDF (Term Frequency - Inverse Document Frequency).
- **Explanation**: Converts text to numerical vectors based on word importance. Words that appear frequently in a message but rarely across all messages get a higher score.

## Slide 6: Model Development
- **Algorithm Used**: Logistic Regression.
  - *Why?* It is simple, interpretable, and performs exceptionally well on binary text classification tasks.
- **Validation Strategy**: K-Fold Cross Validation (k=5).
  - Ensures the model isn't overfitting to a specific train/test split.

## Slide 7: Performance Evaluation & Metrics
- Explain the key metrics calculated on the test set:
  - **Accuracy**: Overall correctness.
  - **Precision**: Out of all predicted spams, how many were actually spam? (Crucial to avoid filtering out important non-spam messages).
  - **Recall**: Out of all actual spams, how many did we catch?
  - **F1-Score**: Harmonic balance between precision and recall.

## Slide 8: Confusion Matrix Analysis
- *Insert `confusion_matrix.png` generated by the code.*
- Explain the four quadrants:
  - True Positives (Correctly identified spam)
  - True Negatives (Correctly identified ham)
  - False Positives (Ham classified as spam)
  - False Negatives (Spam classified as ham)

## Slide 9: System Architecture
- **Frontend**: HTML, Tailwind CSS, Vanilla JS. Features Dark Mode and responsive UI.
- **Backend**: FastAPI. Handles API requests, loads `.pkl` models, and processes predictions.
- **Communication**: REST API using the `fetch` protocol.
